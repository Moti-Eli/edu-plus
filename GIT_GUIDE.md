
===========================================
Git לעבודה עם Claude Code - קיצור
===========================================

לפני שינוי:
git add . && git commit -m "backup"

אם לא טוב - חוזר אחורה:
git checkout .

אם טוב - שומר:
git add . && git commit -m "תיאור השינוי"

# מדריך Git לפרויקט

## 🔵 פקודות בסיסיות

### לראות מה השתנה
```bash
git status
```
**מתי:** בכל פעם שרוצים לראות אילו קבצים השתנו מאז השמירה האחרונה

---

### לראות בדיוק מה השתנה בתוך הקבצים
```bash
git diff
```
**מתי:** כשרוצים לראות את השורות שהשתנו (ירוק = נוסף, אדום = נמחק)

---

## 🟢 שמירת שינויים (Commit)

### שלב 1: הוספת קבצים לשמירה
```bash
git add .
```
**מתי:** לפני כל commit - מוסיף את כל השינויים

### שלב 2: שמירת נקודת גיבוי
```bash
git commit -m "תיאור השינוי"
```

**דוגמאות:**
```bash
git commit -m "הוספתי טבלת מערכת שעות"
git commit -m "תיקון באג בדף התחברות"
git commit -m "עיצוב חדש לדף מנהל"
git commit -m "backup before AI changes"
```

### קיצור - שני השלבים ביחד:
```bash
git add . && git commit -m "תיאור השינוי"
```

---

## 🔴 ביטול שינויים (חזרה אחורה)

### ביטול כל השינויים שלא נשמרו
```bash
git checkout .
```
**מתי:** משהו השתבש ורוצים לחזור ל-commit האחרון
**⚠️ זהירות:** מוחק את כל השינויים מאז השמירה האחרונה!

### ביטול שינויים בקובץ ספציפי
```bash
git checkout שם-הקובץ
```
**דוגמה:**
```bash
git checkout components/Chat.tsx
git checkout app/admin/page.tsx
```

### ביטול קובץ שנוסף ל-staging (אחרי git add)
```bash
git reset HEAD שם-הקובץ
```

---

## 📜 היסטוריה

### לראות רשימת כל השמירות
```bash
git log --oneline
```
**דוגמת פלט:**
```
a1b2c3d הוספתי AI לצ'אט
e4f5g6h תיקון באג בטבלה
1cdf20d backup before changes
```

### לראות היסטוריה מפורטת
```bash
git log
```

---

## ⏪ חזרה לגרסה קודמת

### לראות איך נראה קובץ בגרסה קודמת
```bash
git show HEAD~1:שם-הקובץ
```
**דוגמה:** לראות איך Chat.tsx נראה לפני commit אחד:
```bash
git show HEAD~1:components/Chat.tsx
```

### לחזור commit אחד אחורה (זהירות!)
```bash
git reset --hard HEAD~1
```
**⚠️ זהירות:** מוחק את ה-commit האחרון לגמרי!

---

## 🔄 תהליך עבודה מומלץ

### לפני שמתחילים לעבוד על פיצ'ר חדש:
```bash
git status                              # לוודא שהכל נקי
git add . && git commit -m "backup"     # לשמור מצב נוכחי
```

### אחרי שסיימתם פיצ'ר:
```bash
git status                              # לראות מה השתנה
git add .                               # להוסיף הכל
git commit -m "תיאור הפיצ'ר"            # לשמור
```

### אם משהו השתבש:
```bash
git checkout .                          # לחזור לשמירה האחרונה
```

---

## 📋 סיכום פקודות

| פקודה | מה עושה |
|-------|---------|
| `git status` | מראה מה השתנה |
| `git diff` | מראה את השינויים בפירוט |
| `git add .` | מוסיף הכל לשמירה |
| `git commit -m "הודעה"` | שומר נקודת גיבוי |
| `git checkout .` | מבטל את כל השינויים |
| `git checkout קובץ` | מבטל שינויים בקובץ ספציפי |
| `git log --oneline` | מראה היסטוריית שמירות |

---

## 💡 טיפים

1. **תמיד תעשו commit לפני שינויים גדולים** - ככה קל לחזור אחורה
2. **כתבו הודעות commit ברורות** - תודו לעצמכם בעתיד
3. **אל תפחדו מ-git checkout .** - זה פשוט חוזר לשמירה האחרונה
4. **עשו commit לעתים קרובות** - הרבה נקודות שמירה = הרבה אפשרויות לחזור

---

## 🆘 עזרה

אם משהו לא עובד:
```bash
git status      # קודם כל לראות מה המצב
```

אם יש קונפליקטים או בעיות:
```bash
git checkout .  # לנקות הכל ולהתחיל מחדש
```
